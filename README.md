RecipeFinder — це десктопний застосунок на базі .NET/WPF, створений для організації кулінарного досвіду користувача. 
Проєкт демонструє застосування принципів Clean Code, архітектури MVVM та сучасних патернів проєктування.
Функціональні можливості
 - Управління рецептами (CRUD): Додавання, перегляд, оновлення та видалення рецептів.
 - Розумний пошук: Динамічна фільтрація бази за назвою за допомогою гнучких стратегій.
 - Валідація даних: Багаторівнева перевірка введених даних (назви, інгредієнтів, кроків).
 - Конвертація та масштабування: Перерахунок ваги/об'єму та зміна кількості порцій.
 - Експорт у файл: Збереження рецепта у структурований текстовий формат (.txt) для друку.
 - Локальне сховище: Автоматична робота з даними у форматі JSON

Programming Principles (Принципи програмування)
 - У проєкті дотримано мінімум 5 принципів чистого коду:
 - Single Responsibility Principle (SRP): Кожен клас відповідає за одну задачу. Наприклад, ExportService.cs лише генерує файли, а ValidationService.cs лише перевіряє дані.
 - Encapsulation: Стан об'єктів прихований; доступ до даних здійснюється через публічні властивості з валідацією.
 - DRY (Don't Repeat Yourself): Спільна логіка сповіщення UI та математичні обчислення винесені в окремі сервіси та базові класи.
 - KISS (Keep It Simple, Stupid): Архітектура інтерфейсів побудована максимально просто для полегшення розуміння та підтримки.
 - Dependency Inversion: Високорівневі модулі (ViewModel) залежать від абстракцій (IRecipeRepository), а не від конкретних реалізацій.

 Design Patterns (Патерни проєктування)
Згідно з вимогами, реалізовано 3 патерни:
 - MVVM (Model-View-ViewModel): Повне розділення бізнес-логіки та UI частини.
Repository (Сховище):
 - Файли: IRecipeRepository.cs, JsonRecipeRepository.cs.
 - Призначення: Ізоляція логіки роботи з JSON-файлами від решти програми.
Strategy (Стратегія):
 - Файли: ISearchStrategy.cs, NameSearchStrategy.cs.
 - Призначення: Дозволяє динамічно змінювати алгоритми фільтрації рецептів без зміни коду клієнта.
	
Refactoring Techniques (Техніки рефакторингу)
Список застосованих технік для покращення структури коду:
 - Extract Method: Складна логіка перевірки інгредієнтів винесена в метод ValidateIngredients у класі ValidationService.cs.
 - Rename Method: Використано чіткі назви за стандартом PascalCase (наприклад, ExportToText, ScaleAmount).
 - Extract Class: Логіка валідації та конвертації винесена з ViewModel у сервіси ValidationService та UnitConverter.
 - Move Method: Розрахунок калорій та масштабування порцій перенесено до бізнес-логіки моделей та сервісів.
 - Replace Magic Number: Замість числових значень у коді впроваджено константи GramsInOunce та MlInCup у UnitConverter.cs.
